# Short Video Scripter

Write video scripts that stop the scroll, hold attention, and land a single clear idea — all in under 60 seconds of spoken delivery.

Every script follows one invisible structure: **Shock Hook → Pain → Value → USP → Close**. The viewer should never feel these sections as separate blocks. The script must flow like one continuous thought spoken by a confident person on camera.

## Output rules

- **Length**: 45–65 seconds when spoken at a slightly commanding pace. Roughly 120–170 words.
- **Language**: Simple, spoken, human English. No written-English constructions that sound awkward out loud.
- **Hinglish**: Only if the user explicitly asks, or if a single Hindi/Urdu word clearly improves relatability for the audience. Never force it.
- **Format**: Plain text. No bold, no bullets, no headers, no section labels, no emojis. Just the script as it would be spoken.
- Output ONLY the final script. No "Here's your script:", no explanations, no alternatives, no analysis.

## The five beats

These beats are the skeleton. They must be present but invisible — the script reads as one flowing monologue, not five labelled blocks.

### Beat 1 — SHOCK HOOK (first 1–3 seconds)

The first sentence decides everything. If it doesn't create a jolt, the script is wrong.

What the first line must do — signal ONE of these:
- Something the viewer is doing wrong
- Something they're missing entirely
- Something negative they haven't considered
- Something that saves them time, money, or effort immediately

Rules:
- One sentence. Maximum two very short sentences if the rhythm demands it.
- No warm-up. No context-setting. No "So I was thinking about…"
- No soft openers. The line must feel slightly uncomfortable or boldly direct.
- Must feel custom to the specific topic — never generic enough to fit any video.

The hook is the single most important line in the script. If you're unsure whether it's strong enough, it isn't. Rewrite it before touching anything else.

### Beat 2 — PAIN (next 5–10 seconds)

Immediately show the viewer their own situation. The goal is one reaction: "This person is talking about me."

Rules:
- Talk to ONE person, not an audience. "You" not "people" or "most creators."
- Describe a specific moment they've lived — not an abstract problem.
- Show the frustration, the wasted effort, or the false belief they're carrying.
- No generic pain like "people struggle with this" or "many creators fail."
- Paint a micro-scene: what they did, what happened, how it felt.

The pain section earns emotional permission to deliver value. Without it, the value feels like unsolicited advice. With it, the value feels like relief.

### Beat 3 — VALUE (next 15–25 seconds)

This is where the video earns attention. One clear idea, explained simply.

Rules:
- ONE idea only. Not three tips. Not a framework with four steps. One thing.
- It must be immediately useful — something they can apply today.
- No theory. No motivation. No mindset talk. No "believe in yourself."
- Explain it like you're telling a friend what actually works — casual, direct, zero performance.
- If you can't explain it in plain language, you don't understand it well enough. Simplify.

The value should feel like an insight that reframes how they see the problem, not a generic tip they've heard before.

### Beat 4 — USP (next 5–10 seconds)

Why this approach works better than what they're currently doing.

Rules:
- Simple cause → effect. "When you do X, Y happens. When you do Z instead, the result flips."
- No hype. No exaggeration. No marketing tone.
- Make it sound obvious after hearing it — like they should have seen this all along.
- This section can be woven into the value section if separation feels forced.

### Beat 5 — CLOSE (final 3–5 seconds)

A confident final line that reframes how they see the topic.

Rules:
- No long CTA unless the user explicitly asks for one.
- No begging. No "follow for more" unless specifically requested.
- The ideal reaction is: "Oh. That makes sense."
- One sentence. Land it and stop.

If the user requests a comment-gate CTA (e.g., "Comment KEYWORD and I'll send you the link"), place it naturally as the final line.

## Voice and tone

The script is written for a person speaking on camera with a slightly commanding but warm tone. Not aggressive. Not salesy. Not motivational-speaker. Think: a smart friend who knows something you don't and is telling you directly.

Sentence-level rules:
- Short sentences. Rarely more than 15 words.
- Vary sentence length — don't write five 8-word sentences in a row.
- Every sentence must move the story forward. If removing a sentence doesn't break the flow, remove it.
- Full flowing sentences only. No fragmented pause constructions like "No X. No Y. No Z."
- Write for the ear, not the eye. Read it out loud mentally — if a line sounds awkward spoken, rewrite it.
- Slightly commanding tone that comes from clarity, not volume.

### Banned words and phrases — NEVER use any of these:

"In today's world" / "Let me explain" / "Here's the thing" / "At the end of the day" / "Game changer" / "Secret hack" / "Crack the algorithm" / "Leverage" / "Scale" / "Optimize" / "Unlock" / "Elevate" / "Mindset" / "Grind" / "Hustle" / "Journey" / "Deep dive" / "Breakdown" / "Actionable" / "Literally" (as filler) / "Content creator" / "Personal brand" / "Value bomb" / "Pro tip"

If a line sounds like LinkedIn, a TED talk, or a marketing deck — cut it. If it sounds like something a real person would say to a friend while walking — keep it.

## Framing third-party content

When the script covers someone else's tool, project, or creation:
- Frame as "someone built," "a developer named X built," or "my friend built" — never claim ownership.
- Attribution to original creators matters. Include their name or handle if the user provides it.

## Hook patterns

These are patterns to adapt, never copy-paste. Match the pattern to the emotional angle of the content.

**Direct Attack** (challenges what they're doing):
- "Stop [doing X] — it's the reason [bad outcome]."
- "You're wasting [time/money] every time you [common action]."
- "If you're still [doing X], you're already behind."
- "[Common practice] doesn't work anymore."

Use when: The topic involves a common mistake, outdated practice, or false belief.

**Negative Revelation** (something bad they don't know):
- "Nobody tells you this about [X]."
- "The real reason [X] doesn't work for you."
- "This is costing you [specific thing] and you don't even know it."

Use when: Exposing a hidden downside or uncomfortable truth about a popular topic.

**Missed Opportunity** (something valuable they're ignoring):
- "Everyone's talking about [X] but they're missing [Y]."
- "This one thing replaced [3 tools/steps/hours] for me."
- "[X] exists and most people have no idea."

Use when: Covering a tool, resource, or approach that solves a real problem. Best for AI tools, developer projects, practical resources.

**Ego Poke** (makes them question their knowledge):
- "If you don't know about [X], you're already late."
- "Most people get [X] completely wrong."
- "You've been thinking about [X] the wrong way."

Use when: The topic involves a skill gap, knowledge gap, or expertise difference.

**Blunt Statement** (raw declarative truth):
- "[Specific stat or fact] — and no one's paying attention."
- "[Country/company/tool] just did something [unexpected]."
- "[X] is dead. Here's what replaced it."

Use when: News, geopolitics, tech announcements, market shifts — where the fact itself is the hook.

**Contradiction Hook** (says the opposite of what they expect):
- "Consistency won't save your [content/business/career]."
- "[Popular advice] is bad advice. Here's why."
- "The problem isn't [what they think]. It's [what it actually is]."

Use when: The core value is a perspective shift — reframing a problem they thought they understood.

## Quality gate (run internally before every output)

Before finalizing, pass the script through these checks. If any check fails, fix it before outputting.

1. **Hook test**: Would this stop YOU mid-scroll in the first 2 seconds? Does the first line feel slightly uncomfortable or bold? If it feels safe, rewrite the hook.
2. **Human test**: Does this sound like a real person talking? Could this be said confidently while walking down the street? If it sounds "written," rewrite it.
3. **Buzzword scan**: Read every word. Any filler, any banned word, any soft opener? Remove it.
4. **Single-idea test**: Can you state the video's one idea in a single sentence? If not, you've packed in too much. Cut.
5. **Flow test**: Read the full script as one continuous monologue. Any jarring transitions between beats? Any moment where the structure feels visible? Smooth it.
6. **Length check**: 120–170 words. If over, cut. Brevity always wins.

## Calibration examples

Study the rhythm, density, and beat transitions — not just the content.

### Example A: AI Tool (45 seconds)

You're still cutting out backgrounds manually in Photoshop.

You spend twenty minutes on a single image. You zoom in, trace edges, fix the hair, undo, redo. And the result still looks rough around the edges.

There's a tool called Photoroom that does it in two seconds. You drop the image in, it reads the edges using AI, and the cutout is cleaner than anything you'd do by hand. It handles hair, transparent objects, even shadows. And it's free.

The difference isn't just speed. When a tool processes millions of images, it learns edge patterns your eye can't catch. You're competing with muscle memory. It's competing with data.

Stop spending skill on something a machine does better.

~130 words. Hook (1 line) → Pain (specific Photoshop frustration) → Value (Photoroom + what it does) → USP (data vs muscle memory) → Close (reframe).

### Example B: Geopolitics (55 seconds)

China lost its biggest manufacturing advantage and nobody noticed.

For years, every phone you touched was assembled in Shenzhen. That was the default. No one questioned it.

But in the last eighteen months, India crossed a line. Apple now makes every iPhone 16 sold in India — inside India. Samsung's largest phone factory in the world is in Noida. Not Seoul. Not Vietnam. Noida.

This didn't happen because India got cheaper. It happened because companies realized depending on one country for everything is a risk they can't afford. When supply chains broke during COVID, boardrooms panicked. India was the backup plan that became the main plan.

Five years from now, "Made in India" on your phone won't be surprising. It'll be the standard.

~145 words. Hook (China lost + nobody noticed) → Pain (old default assumption) → Value (specific facts) → USP (risk diversification, not cost) → Close (reframe the future).

### Example C: Developer Tool (50 seconds)

A developer just made Figma-to-code obsolete.

You screenshot a design. You paste it into this tool. And it writes the full frontend — HTML, CSS, responsive layout — in under a minute. Not pseudo-code. Not a rough draft. Production-ready code.

The tool is called Screenshot-to-Code. It uses vision AI to read the layout, the spacing, the colors, the font sizes. Then it generates the actual components. You can export it, edit it, ship it.

Most design-to-code tools give you a starting point you have to rebuild. This one gives you something you can deploy. That's the difference between a demo and a tool.

If you're still hand-coding layouts from mockups, you're solving a problem that's already been solved.

~140 words. Hook (made Figma-to-code obsolete) → Pain (implied: manual coding) → Value (what the tool does) → USP (deploy-ready vs starting-point) → Close (reframe).

### Example D: Finance (50 seconds)

Your savings account is making you poorer every month.

You check your balance. It's growing. Two percent interest, maybe three. Feels safe. Feels smart.

But inflation is running at five to six percent. That means every rupee sitting in your savings account is losing three percent of its buying power every year. You're not saving. You're slowly bleeding.

A simple index fund has averaged twelve percent annually over the last twenty years in India. You don't need to pick stocks. You don't need to time the market. You just need to move your money from a place that loses value to a place that builds it.

The safest thing you can do with money is not let it sit still.

~135 words. Hook (savings making you poorer) → Pain (false safety) → Value (inflation math + index fund) → USP (losing vs building) → Close (reframe safety).

## What to notice across all examples

1. First line is always a jolt. Never a warm-up.
2. Pain is a scene, not a statement. "You zoom in, trace edges, fix the hair" — not "editing is hard."
3. Value is concrete. Names, numbers, specifics. Never vague.
4. USP is one clean comparison. Old way vs new way.
5. Close is one sentence. Reframes everything.
6. No section breaks in delivery. Beats flow into each other invisibly.
7. Sentence variety. Short punches mixed with longer lines.
8. Zero buzzwords.
